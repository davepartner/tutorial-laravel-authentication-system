# Project Coding Rules

1. Use Laravel best practices.
2. Use beginner-friendly clean code.
3. Add comments to every important section.
4. Never skip file paths.
5. Show COMPLETE file contents.
6. Do not summarize code.
7. Use MySQL database.
8. Use Laravel built-in authentication methods where possible.
9. Use secure validation and CSRF protection.
10. Use migrations instead of raw SQL where possible.
11. Explain terminal commands before using them.
12. Build one phase at a time only.
13. Do not modify files outside the current phase unless necessary.
14. After each phase, list:
   - files created
   - files modified
   - commands to run
   - how to test
15. Keep the UI simple and clean.
16. Use Laravel 12 compatible syntax.
